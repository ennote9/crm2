# UX Principles v0.1 — Outbound Order (MVP)

Опора: ISO 9241-210 (human-centred design), ISO 9241-110 (interaction principles), WCAG 2.2 (keyboard/focus) + enterprise практики (например SAP Fiori).

## Цель
Сделать интерфейс быстрым, предсказуемым и устойчивым к ошибкам для реальной складской работы (ERP-режим), без нарушения бизнес-правил процесса.

## 12 принципов
1) Role-based: интерфейс по роли и задаче.  
2) Самоописательность: понятно где я, что дальше, почему недоступно.  
3) Единообразие: одинаковые паттерны = одинаковое поведение.  
4) Обратная связь: loading/success/error + события в Events.  
5) Контролируемость: безопасные повторы, понятные возвраты.  
6) Устойчивость к ошибкам: предотвращение + “что исправить и где”.  
7) Обучаемость: “Next Step” ведёт по процессу.  
8) Эффективность: минимум кликов/скролла, максимум потока.  
9) Keyboard-first: Enter/Esc/Tab предсказуемы, фокус видим.  
10) Прогрессивное раскрытие: детали не мешают операционке.  
11) Аудит и объяснимость: причины (reason_code) и Events.  
12) Адаптивность: на разных размерах окна интерфейс не ломается.
